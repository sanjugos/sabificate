## Overview

Step-by-step walkthrough of the mobile learner experience, validating the 5-block lesson model in a functional prototype.

## Course Flow (Maps to 5-Block Model)

### Introduction
1. Navigate to the course landing page
2. Click **"Start course →"**

### Block 1: Scenario
3. **Play audio** version of the workplace scenario
   - Audio playback button for commute/hands-free learning
   - Nigerian workplace context (company names, regulatory bodies, cultural situations)

### Block 2: Knowledge Check
4. Click **"Continue →"** to progress
5. **Select required elements** (e.g., "three elements required for every follow-up message")
   - Interactive selection quiz testing comprehension of the scenario

### Block 3: Core Teaching
6-7. Progress through core teaching content with **"Continue →"** buttons
   - Concise content delivery focused on the learning objective
   - Builds on what the scenario introduced

### Block 4: Application Quiz
8. **Select correct answer** (e.g., "the correct opening for a reliable customer's reminder")
9. Click **"Continue →"**

### Block 5: Mastery & Action Task
10. **Sequence ordering** — arrange steps A, B, C in correct order
    - Tests transfer, not just recall
11. Click **"Continue →"**
12. **Complete action task** and click **"Submit & get credential"**
    - Practical artifact submission that demonstrates applied skill

### Certification
13. **Download PDF** certificate of completion
14. Click **"Browse more courses"** to continue learning

## Key UX Observations

- **Card-swipe interaction pattern** — each block is a discrete card, swiped to progress
- **Audio-first option** — scenario block offers audio playback, critical for commute learning
- **Progressive difficulty** — knowledge check → application quiz → mastery task
- **Immediate credential** — PDF download available right after completion
- **Mobile-optimized** — all interactions designed for thumb navigation on 5-inch screens
- **Minimal data usage** — text-first with optional audio, no video
